Day 1 - Full Stack & SDLC

Date: {{YESTERDAY_DATE}}

What is Full-Stack Development?

Full-stack development means working on both the front-end and back-end of a website or web app.

Front-End

  • User Interface (what the user sees)
  • HTML, CSS, JavaScript, React

Back-End

  • Server-side logic
  • Node.js, Express.js
  • Databases like MongoDB

Full-Stack Developer

A developer who can build both the UI and the backend of an application.

SDLC - Software Development Life Cycle

SDLC is the step-by-step process used to build software correctly and efficiently.

Phases of SDLC

  • Requirement Analysis: Understanding what needs to be built.
  • Planning: Time, cost, tools, and team planning.
  • Design: UI design + system architecture.
  • Development: Writing the code.
  • Testing: Fixing bugs and checking for errors.
  • Deployment & Maintenance: Launching and updating software.

Day 2 - Version Control, Git & GitHub

Date: {{TODAY_DATE}}

What is Version Control?

Version control helps developers track changes, save different versions of a project, and avoid losing work.

What is Git?

Git is a tool used to track changes in code, manage versions, create commits, and work safely on projects.

What is GitHub?

GitHub is an online platform where people upload Git-based projects, collaborate, create branches, and store code.

How Git & GitHub Help a Team of 5 Developers

  • All team members can work on the same project at the same time
  • No one’s code gets overwritten
  • Branches help each person work on separate features
  • Merging combines everyone’s work smoothly
  • GitHub stores the project online safely

Today's Goals

  • Create GitHub account
  • Install Git
  • Create repository
  • Commit and track changes
  • Add files to repository
  • Use VS Code
  • Link VS Code with GitHub
  • Post a website on GitHub
  • Create branches
  • Merge branches
  • GitHub collaboration
